# UberCraft
Minecraft Modpack with a focus in expanding every facet of the game, mod diversity and cross-compatability.

Modded Minecraft often has the issue of making the game too easy, some modpacks try to fix this, for example, by gating progression, this, however, doesn't fix the issue. The issue is that most mods make minecraft's core gameplay easier, gating progression only increases the game's longevity arteficially.

This modpack aims to increase difficulty naturally, that is, by allowing the mods to make the game, in fact, easier, but by ramping up Minecraft's core gameplay difficulty overall. It also aims to expand on Minecraft's core gameplay instead of adding new mechanics, so, none of that going around collecting tweed, rocks and sticks and crafting outside a crafting table that many modpacks do.

Players should interact with all of the games systems to some extent, but they shouldn't be forced to do so extremely often if they dislike a particular system.
In that spirit, this modpack aims to expand all Minecraft's systems, the player will have to engage with most of them from time to time, but only regularly with the ones they choose.

The modpack expands on all of Minecraft's systems, but if you choose to fully engage with one, you'll have drawbacks on others. For example, if someone chooses to fully engage the RPG part of Minecrfat, they should have more trouble gathering resources and breaking blocks, as to not 'cheat' the RPG experience. Players who choose to engage with the more technical aspect should be frailer than average as to not 'cheat' the art of making complex machinery by gathering a bunch of loot. And so on and so forth. 

Gating will, of course, still be present, it's a necessary evil, but only for things such as Waila, the minimap and player's stats. Any additional gating should arrise naturally from what the player can and can't craft.

# Outline

Goal: To create a kitchen-sink expert mode for modded minecraft with a nice difficulty curve and imporved game flow, all while putting the survival mode back in minecraft.

1. Establishing a semi-final modlist, this includes purging redundant or unnecessary mods;
2. Making sure the modpack is stable;
3. Making it so that orespawn is handled by the least amount of mods possible;
4. Defining gamestyles and implementing their bonuses and drawbacks;
5. Removing, adding, changing recipes;
  5.1. Example: a lot of mods have their own version of what a circuit is, unfying this between them would be the goal;
  5.2. A lof of mods have recipes that wouldn't be easy in vanilla Minecraft, but this is modded Minecraft, so they have to be      changed (here's looking at draconic evolution);
  5.3. Have mods use items from other mods as crafting or progression components, but mostly for that specific mod's early game, people shouldn't be forced to go end-game with a mod they hate in order to use one they love;
6. Making it so that only books and guides the player will use early game spawn, and making it so that they spawn inside an Akashik Tome;
7. Defining and implementing which mobs should only appear mid or late game and how mob difficulty progresses;
8. Locking certain features such as the minimap, Hwyla and some dimensions behind certain requirements or achievments;
9. Changing how compat skills works entirely and making it work with most mods;
10. Making a quest/quide book for the player to follow.
